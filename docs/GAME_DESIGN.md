## 3. 温度系统（重写：平衡模型）

### 3.1 设计目标

- 温度**不会无限累加**——它朝一个"稳态温度"收敛
- 玩家优化的是**稳态值够不够低**，而不是"每 30 秒看一眼温度"
- 加卡/关卡的反馈是**渐进的**（30-60 秒到达新稳态），给玩家反应窗口
- 完全挂机友好：离线时温度直接跳到稳态，不会爆炸

### 3.2 核心公式

```
equilibrium = ambient + max(0, total_heat_output - cooling_power) / dissipation_rate

temp(t) = temp(t-1) + (equilibrium - temp(t-1)) × approach_speed × delta_time
```

| 参数 | 含义 | 典型值 |
|---|---|---|
| `ambient` | 房间环境温度 | 城中村 30°C, 北极 −10°C |
| `total_heat_output` | 所有显卡 + PSU 的总发热量 | Σ(card.heat) + Σ(psu.heat) |
| `cooling_power` | 散热设备总功率 | 风扇 3, 液冷 8, 海水冷却 15 |
| `dissipation_rate` | 房间散热系数（越高越容易散热） | 城中村 0.15, 海上 0.40, 北极 0.50 |
| `approach_speed` | 趋近速度 | 0.03（约 30-60 秒到 90% 稳态） |

**示例**：城中村，3 张卡（总热量 9）+ 1 台 PSU（热量 2），1 个风扇（cooling 3）

```
equilibrium = 30 + max(0, 11 - 3) / 0.15 = 30 + 53.3 = 83.3°C  ← 危险！
```

加一台风扇（cooling → 6）：
```
equilibrium = 30 + max(0, 11 - 6) / 0.15 = 30 + 33.3 = 63.3°C  ← 高温区但可控
```

换到海上集装箱（ambient 18, dissipation 0.40）：
```
equilibrium = 18 + max(0, 11 - 6) / 0.40 = 18 + 12.5 = 30.5°C  ← 凉爽
```

### 3.3 温度区间与效果

| 温度区间 | 状态 | 效果 |
|---|---|---|
| < 40°C | 🟢 **凉爽** | 无影响。佣兵心情 +1/h |
| 40-60°C | 🟡 **温热** | 无性能影响。故障率 +2%/°C（即 60°C 时故障率 +40%） |
| 60-80°C | 🟠 **高温** | 效率 −1%/°C（60°C = −0%, 80°C = −20%）。噪音 → 邻居投诉概率上升 |
| 80-95°C | 🔴 **过热** | 效率 −2%/°C。显卡随机降频。每分钟 5% 概率一张卡故障 |
| ≥ 95°C | 💀 **临界** | 新卡拒绝开机。每分钟 15% 概率火灾。佣兵拒绝进入 |

**"贪心空间"设计**：60-80°C 高温区故意留给玩家——不投资散热、让温度停在 75°C，效率低 15%，但省下的钱多买两张卡。配合"余热发电"技能，甚至可以**故意跑高温来回收电费**。

### 3.4 动态事件对温度的影响

事件改变的是**稳态目标**，而非直接改温度。玩家看着温度慢慢朝新稳态爬升/下降，有反应时间。

| 事件 | 温度模型影响 | 恢复 |
|---|---|---|
| **市政断电** | cooling_power → 0（风扇停），稳态飙升 | 来电后 cooling 恢复，温度慢降 |
| **风扇故障** | cooling_power 减半 | 维修后恢复 |
| **季节变化** | ambient ±10°C | 季节切换 |
| **超频** | 被超频卡 heat_output ×2 | 取消超频恢复 |
| **液冷泄漏** | cooling_power → 0 + 短路风险 | 派 IT 佣兵修理 |
| **PSU 过载发热** | PSU heat_output ×3 | 降载或换 PSU |

### 3.5 离线处理

玩家关闭游戏后：
- 温度直接设为当前配置的 `equilibrium`（跳过趋近过程）
- 离线期间故障判定按稳态温度对应的故障率计算
- 回来时"喵日志"只报告稳态下发生的事件，不会出现"离线时温度爆炸"

### 3.6 实现代码（Godot）

```gdscript
# room.gd
var ambient: float
var dissipation: float
var cooling: float       # 散热设备总功率（可升级）
var current_temp: float

const APPROACH_SPEED := 0.03

func get_total_heat() -> float:
    var heat := 0.0
    for card in active_cards:
        heat += card.heat_output
    for psu in psu_units:
        heat += psu.heat_output
    return heat

func get_equilibrium() -> float:
    return ambient + max(0.0, get_total_heat() - cooling) / dissipation

func _process(delta: float) -> void:
    var eq := get_equilibrium()
    current_temp += (eq - current_temp) * APPROACH_SPEED * delta
    apply_temp_effects(current_temp)
```

## 4. PSU 电源系统（新增）

### 4.1 设计意图

现实中矿场最常烧的不是显卡而是电源。PSU 做成独立设备后，给电力管理加了一层**硬约束**：不是有钱买卡就能用，还得有足够的 PSU 功率来带动。

### 4.2 PSU 属性

| 字段 | 说明 |
|---|---|
| `name` | 型号名 |
| `rated_power` | 额定功率（W）——能带多少显卡的硬上限 |
| `efficiency` | 转换效率（80%/85%/90%）——实际耗电 = 显卡总耗电 / efficiency |
| `heat_output` | PSU 自身发热（计入温度系统） |
| `quality` | 品质等级（Bronze / Silver / Gold / Platinum） |
| `overload_tolerance` | 过载容忍度（超过额定功率多少%才炸） |
| `price` | 购买价格 |
| `explosion_damage` | 炸了波及多少设备 |

### 4.3 PSU 图鉴

| 名称 | 额定功率 | 效率 | 发热 | 品质 | 过载容忍 | 价格 | 备注 |
|---|---|---|---|---|---|---|---|
| **垃圾桶 PSU** | 300W | 75% | 4 | 无 | 5% | $30 | 超过 305W 直接炸，波及相邻 2 张卡 |
| **杂牌 500W** | 500W | 80% | 3 | Bronze | 10% | $80 | 基础款 |
| **海韵 650W** | 650W | 85% | 2 | Silver | 15% | $200 | 性价比之选 |
| **酷冷至尊 850W** | 850W | 88% | 2 | Gold | 20% | $450 | 中期主力 |
| **海盗船 1200W** | 1200W | 90% | 1.5 | Gold | 25% | $900 | 后期大型矿场用 |
| **服务器级 1600W** | 1600W | 92% | 1 | Platinum | 30% | $2,000 | 适合 A100 |
| **MEOWCore PSU（自研）** | 2000W | 95% | 0.5 | Custom | 40% | 研发 | 需要工程线顶级 + R&D |

### 4.4 PSU 核心机制

**硬约束**：每个房间可安装 1-4 台 PSU（后期升级可更多）。房间内所有显卡的 `power_draw` 总和不能超过所有 PSU 的 `rated_power` 总和。

```
total_card_power = Σ(card.power_draw)
total_psu_capacity = Σ(psu.rated_power)

if total_card_power > total_psu_capacity:
    不允许开机更多显卡（灰色"超载"提示）

if total_card_power > total_psu_capacity × (1 + psu.overload_tolerance):
    → 触发 PSU 爆炸事件
```

**实际电费计算（PSU 效率影响）**：

```
actual_power_consumed = total_card_power / avg_psu_efficiency
cost_per_sec = actual_power_consumed × room.electric_cost × 0.0017
```

低效率 PSU（75%）意味着：显卡吃 100W，实际从墙上抽 133W，多出的 33W 全变成热量。**劣质 PSU 既费电又费热**。

**PSU 发热计入温度系统**：
```
total_heat = Σ(card.heat_output) + Σ(psu.heat_output) + wasted_power_as_heat
wasted_power_as_heat = total_card_power × (1/avg_efficiency - 1) × 0.1
```

### 4.5 PSU 的决策张力

- **早期困境**：$150 启动资金，一张 1060（$120）+ 一台垃圾 PSU（$30）= 刚好。但垃圾 PSU 只能带 300W，再买第二张卡就超载了。**玩家的第二次购买不是显卡，而是 PSU**。
- **效率 vs 价格**：Bronze 80% 便宜但浪费电+发热多，Gold 90% 贵但长期省钱。跟显卡的"超频 vs 节能"改装路线形成交叉决策。
- **爆炸风险**：劣质 PSU 过载容忍低。玩家可以冒险在 300W PSU 上跑 310W（5% 容忍内），但如果有显卡超频升级后 power_draw 涨了，可能悄悄超过阈值——**PSU 爆炸是静默的，不会提前警告**，只有在超过阈值时有概率触发。
- **连锁损失**：PSU 爆炸不只是自己坏——波及相邻显卡。垃圾 PSU 炸了带走 2 张卡，服务器级只损失自己。

### 4.6 PSU 升级与维护

- PSU **不可升级**，只能替换——拔旧的装新的
- 替换 PSU 需要**关闭该 PSU 上所有显卡**（停机 2 分钟）
- 旧 PSU 可卖回 30% 原价，或拆解得 1 个研究碎片
- 技能树工程线 `Wiring Optimization` 节点：PSU 过载容忍 +10%
- 自研 MEOWCore PSU 需要工程线到顶 + 额外蓝图

## 5. 矿池系统（新增）

### 5.1 设计意图

现实中个人矿工几乎不可能 solo 挖到一个块。矿池系统让"怎么挖"成为跟"挖什么"同等重要的决策。

### 5.2 两种挖矿模式

#### Solo 挖矿
```
每 tick 判定：
  概率 = player_hashrate / total_network_hashrate
  成功 → 获得整个区块奖励（极大值）
  失败 → 获得 0
```

- 期望收益与矿池相同，但**方差极大**
- 早期（算力占比 0.2%）可能连续 10 分钟一毛不挣，然后突然中一块 = 几千美元
- **彩票心态**：适合"赌一把"的玩家，或算力极高（占比 > 5%）时才稳定
- 无手续费

#### 矿池挖矿
```
每 tick：
  btc_earned = block_reward × (player_hashrate / pool_total_hashrate) × (1 - pool_fee)
```

- 收益稳定（每秒都有进账）
- 但要交**矿池手续费**（1-5%）
- 矿池本身有风险（见下）

### 5.3 矿池列表

| 矿池 | 手续费 | 结算模式 | 特色 | 风险 |
|---|---|---|---|---|
| **ScratchPool（主流大池）** | 2% | PPLNS | 稳定、人多、抗波动 | 低。大池算力高，份额被稀释 |
| **KittenHash（小池）** | 1% | PPS | 手续费低、人少 | 中。小池运气波动大 |
| **DarkClaw（暗网池）** | 3% | PPLNS | 匿名、不查来源 | 中。被执法盯上概率 +5% |
| **WhiskerFi（DeFi 池）** | 0.5% | PPS+ | 手续费最低 | 高。10% 概率跑路 |
| **Solo** | 0% | — | 零抽成 | 极高波动 |

### 5.4 结算模式解释

| 模式 | 全称 | 机制 | 玩家体验 |
|---|---|---|---|
| **PPS** | Pay Per Share | 每提交一份算力就立刻结算，不管矿池有没有挖到块 | 最稳定，但手续费含矿池风险溢价 |
| **PPLNS** | Pay Per Last N Shares | 矿池挖到块后，按最近 N 份贡献分配 | 稍有波动，但长期公平 |
| **PPS+** | PPS + 交易费分成 | PPS 基础上额外分配区块内交易手续费 | 收益略高但矿池需要承担更多风险（所以跑路概率高） |

### 5.5 矿池切换成本

- 切换矿池需要 **10 分钟过渡期**（旧池结算 + 新池注册）
- 过渡期内**无 BTC 收入**
- PPLNS 池如果中途离开，**未结算的 share 作废**（相当于白干了一段时间）
- 技能树大亨线节点 `Pool Hopping` 可将过渡期缩短到 3 分钟

### 5.6 矿池事件

| 事件 | 触发 | 效果 |
|---|---|---|
| **矿池算力暴涨** | 大量 NPC 矿工加入同一矿池 | 你的份额被稀释，收益 −20%（持续到你换池或 NPC 离开） |
| **矿池挖到大块** | PPLNS 池随机 | 一次性获得正常收益 5-10 倍（PPLNS 的"运气"） |
| **矿池跑路** | WhiskerFi 专属，10% 概率/周 | 矿池关闭，待结算 BTC 全没。触发后该矿池永久消失 |
| **矿池被攻击** | DarkClaw 专属 | 2 小时内收益 −50%（池被 DDoS） |
| **Stale Share 波动** | 网络差的房间 | 5-15% 的算力提交被判过期，等于白算。升级"网络优化"可缓解 |

### 5.7 Solo 挖矿的戏剧性

Solo 模式下，每次"中块"是一个**庆祝事件**：
- 屏幕闪金光 + 猫跳起来 + 特殊音效
- 获得 1 个完整 block_reward（随 Halving 减半）
- 但可能连续 30 分钟不中——期间收入 = 0，电费照扣
- **Solo 挖矿是后期"赌博流"的核心**：算力占比 3%+ 时，平均 5-10 分钟中一块，体验从彩票变成稳定但刺激的节奏

### 5.8 矿池与其他系统的耦合

| 系统 | 耦合关系 |
|---|---|
| **全网算力** | 矿池内的 NPC 矿工构成池总算力，影响玩家份额 |
| **BTC 市场** | PPS 模式下 BTC 价格波动不影响结算（已按算力定价）；PPLNS 模式下价格暴涨时中块 = 大赚 |
| **技能树·大亨线** | `Pool Hopping` 降低切换成本；`金融帝国` 可购买矿池股份（被动分红） |
| **技能树·黑客线** | `DDoS NPC 矿工` 可以打击特定矿池的 NPC 算力 |

| 8 | **近地轨道卫星舱** | 5°C | 0.60（真空辐射） | 25 | 4 | 0.5%（直连卫星网） |

**地下室的陷阱**：ambient 低（22°C）看起来不错，但 dissipation 只有 0.10——热量散不出去。10 张卡（总热量 50）+ 2 台 PSU（热量 6）在地下室：

```
eq = 22 + max(0, 56 - 0) / 0.10 = 22 + 560 = 582°C  ← 显然爆炸
```

加 20 级散热（cooling = 40）：
```
eq = 22 + max(0, 56 - 40) / 0.10 = 22 + 160 = 182°C  ← 还是爆
```

加 50 级散热（cooling = 50）：
```
eq = 22 + max(0, 56 - 50) / 0.10 = 22 + 60 = 82°C  ← 勉强过热区
```

**这就是为什么地下室"必须投资风冷/液冷"**——dissipation 低意味着散热投资的边际收益巨大，但每一点 cooling 都很贵。

### 7.2 Stale Share 率（新增）

不同房间的网络质量影响矿池效率：
- Stale Share = 你提交了算力但因为网络延迟被判"过期"，白算
- 等效于：`effective_efficiency = card.efficiency × (1 - stale_rate)`
- 城中村（8%）：每张卡白丢 8% 算力
- 卫星舱（0.5%）：几乎无损失
- 技能树可解锁"网络优化"节点，所有房间 stale_rate −3%

## 8. 升级、快递与二手市场（修订：BTC 联动价）

### 8.1 核心改动：二手价跟 BTC 价格联动

v2.0 的卖卡回收是固定比例。v2.1 改为**动态定价**——显卡的二手市场价跟 BTC 价格挂钩。

**公式**：

```
btc_price_ratio = current_btc_price / base_btc_price    # base = $30,000

market_multiplier = 1.0 + (btc_price_ratio - 1.0) × card.btc_sensitivity

resale_price = card.price × card.base_resale_ratio × market_multiplier
```

**示例**：

GTX 3080（原价 $5,500，base_resale 0.45，btc_sens 0.8）：

| BTC 价格 | price_ratio | market_mult | 二手价 | 相当于原价% |
|---|---|---|---|---|
| $30,000（基准） | 1.0 | 1.0 | $2,475 | 45% |
| $45,000（牛市） | 1.5 | 1.4 | $3,465 | 63% |
| $15,000（矿难） | 0.5 | 0.6 | $1,485 | 27% |
| $9,000（崩盘） | 0.3 | 0.44 | $1,089 | 20% |

**矿难的双重打击**：
1. BTC 价格跌 → 挖矿收入减少
2. BTC 价格跌 → 显卡二手价也跌 → 你的"资产"缩水

玩家在 BTC $45,000 时花 $5,500 买的 3080，如果 BTC 崩到 $9,000，这张卡只值 $1,089——**浮亏 80%**。这就是真实矿圈的体验。

### 8.2 拆解不受 BTC 影响

- **卖给商店**：受 BTC 联动价影响（动态）
- **拆解**：固定得 20% 原价 + 研究碎片（不受市场影响）

**设计意图**：矿难时"卖卡"很亏，但"拆卡做研究"的价值不变。这鼓励玩家在熊市转向 R&D 路线——正好对应现实中矿难后矿工转行做硬件研发的现象。

### 8.3 购买价格也联动

不只是卖价——**买价也跟 BTC 联动**（幅度较小）：

```
buy_price = card.base_price × (1.0 + (btc_price_ratio - 1.0) × card.btc_sensitivity × 0.3)
```

- 牛市时显卡溢价（黄牛）
- 熊市时显卡打折（矿工抛售）
- **最佳购卡时机 = BTC 暴跌后**——卡便宜，但你的现金也少了（如果你的 BTC 还没卖的话）

### 8.4 "资产净值"面板（新增 UI）

主界面新增"净资产"指标：

```
net_worth = cash
          + btc_held × current_btc_price
          + Σ(card.resale_price)          ← 随 BTC 实时变化
          + Σ(psu.resale_price)           ← PSU 不联动，固定折旧
          - outstanding_debt              ← 如果有贷款/合伙人分红
```

**Prestige 触发条件用 net_worth**：这意味着 BTC 暴涨时你可能突然达到 Prestige 门槛（$1M），但如果等一等可能又跌回去了。**"什么时候退休"变成了一个市场时机判断**。

## 11. 比特币市场（修订）

### 11.1 BTC 产出公式（整合矿池）

```
# Solo 模式
btc_per_tick = 0  或  block_reward    （概率 = player_hash / network_hash）

# 矿池模式
btc_per_tick = block_reward
             × (player_hash / pool_hash)
             × (1 - pool_fee)
             × (1 - stale_rate)

# 通用
effective_btc = btc_per_tick × (1 - gas_fee_rate)
```

### 11.2 Gas Fee（链上手续费，新增）

每次卖 BTC 兑现需要支付链上手续费：

```
gas_fee_rate = base_gas × (1 + network_congestion)
```

- `base_gas` = 0.5%
- `network_congestion` = `total_network_hashrate / 10`（全网算力越高，链越拥堵，手续费越高）
- 典型范围：0.5% - 3%
- 技能树大亨线 `Gas Optimizer` 节点：gas_fee_rate −40%
- **批量卖出**比频繁小额卖出更划算（每次卖出有固定 gas 底费 $5）

### 11.3 减半对矿池的影响（新增）

Halving 后：
- `block_reward` 减半
- 矿池内 NPC 矿工中效率低的 15% 退出 → 你在池内份额临时提升
- 小矿池可能因为算力不足而"死亡"（2 周内不出块 → 池关闭）
- **选大池还是小池在 Halving 前成为关键决策**

## 13. 数值基线（修订）

### 13.1 早期（0-30 min）

- 初始现金 $150
- 初始显卡：1× GTX 1060（120W, 热量 2）
- 初始 PSU：1× 垃圾桶 PSU（300W, 热量 4）
- 初始房间：城中村（ambient 30°C, dissipation 0.15, cooling 0）
- 初始矿池：自动加入 ScratchPool（大池，2% 手续费）
- BTC 初始价 $30,000

**温度校验**：
```
total_heat = 2 (卡) + 4 (PSU) = 6
equilibrium = 30 + max(0, 6 - 0) / 0.15 = 30 + 40 = 70°C  ← 高温区！
```

开局就在高温区（效率 −10%），但不会爆炸。**第一个决策**：花 $50 买个风扇（cooling +3）→ 稳态降到 50°C。但这 $50 也可以省下来攒第二张卡。**trade-off 从第一分钟开始**。

**PSU 校验**：
```
卡耗电 120W，PSU 额定 300W → 余量 180W → 还能带 1 张 1060
第 3 张 1060 = 360W > 300W → 必须升级 PSU
```

**收入校验**：
```
player_hash = 0.0012
pool_hash = 0.5 (NPC) + 0.0012 (你) ≈ 0.5012
btc_per_sec = block_reward × (0.0012/0.5012) × (1 - 0.02) × (1 - 0.08)
            ≈ block_reward × 0.00221

# 假设 block_reward 按缩放后约 = 0.1 BTC/min
btc_per_min ≈ 0.000221 BTC
收入 ≈ 0.000221 × $30,000 = $6.63/min
电费 ≈ 120/0.75 × 1.0 × 0.0017 = $0.272/min
net ≈ $6.36/min
```

30 分钟净赚约 $190 → 够买第 2 张卡 + 可能升级 PSU。节奏合理。

### 13.2 关键公式（汇总）

**温度**：
```
equilibrium = ambient + max(0, total_heat - cooling) / dissipation
temp(t) += (equilibrium - temp(t-1)) × 0.03 × delta
```

**BTC 产出（矿池模式）**：
```
btc = block_reward × (player_hash / pool_hash) × (1 - pool_fee) × (1 - stale_rate)
```

**电费（含 PSU 效率）**：
```
actual_power = Σ(card.power_draw) / avg_psu_efficiency
cost_per_sec = actual_power × room.electric_cost × 0.0017
```

**余热发电**：
```
heat_recovery = (current_temp - ambient) × 0.005 × room.electric_cost
```

**显卡二手价**：
```
btc_ratio = current_btc / 30000
market_mult = 1 + (btc_ratio - 1) × card.btc_sensitivity
resale = card.price × card.base_resale × market_mult
```

**净资产**：
```
net_worth = cash + btc_held × btc_price + Σ(card.resale) + Σ(psu.resale × 0.7)
```

**PSU 过载判定**：
```
if Σ(card.power_draw) > Σ(psu.rated_power × (1 + psu.overload_tolerance)):
    trigger PSU_EXPLOSION event
```

### 13.3 平衡校准点（更新）

## 🐾 v2.1 改动总结

| 改动项 | v2.0 状态 | v2.1 改进 |
|---|---|---|
| 温度系统 | 隐含逐秒累加 | **Newton 冷却平衡模型**：趋近稳态，挂机安全，分 5 个温度区间 |
| PSU | 不存在 | **独立设备**：功率硬约束 + 效率影响电费 + 发热计入温度 + 爆炸风险 |
| 矿池 | 不存在 | **Solo vs 4 种矿池**：手续费/结算模式/跑路风险/份额稀释 |
| 显卡二手价 | 固定比例 | **BTC 联动**：牛市溢价、矿难资产缩水、抄底窗口 |
| 净资产 | 不存在 | **动态净资产**：影响 Prestige 门槛、资产面板实时显示 |
| Stale Share | 不存在 | **房间网络质量**：影响矿池有效算力 |
| Gas Fee | 不存在 | **链上手续费**：全网算力越高越贵 |
| 事件 | 无 PSU/矿池相关 | 新增 8 个事件 + 2 条事件链（PSU 爆炸链、矿难链） |
| 技能树 | 无 PSU/矿池节点 | 新增 6 个节点整合进现有三线 |

> **算力可以堆，电源会先炸；矿池能保底，跑路全白搭；牛市卡值钱，矿难一起塌；温度看稳态，别盯着它爬。**
